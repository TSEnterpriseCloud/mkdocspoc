    Policy di Acceptable Use

    Le seguenti norme, così come le condizioni generali di utilizzo API MadBit, regolano l'utilizzo delle API dei software di titolarità di MadBit Entertainment S.r.l. a socio unico.

    1. Identificazione

    Per accedere ai dati degli utenti di MadBit è necessario fornire agli stessi ed a MadBit informazioni chiare e accurate sull'utilizzo che si farà degli stessi e delle API; ciò include, a titolo esemplificativo:

    • Chi richiede i dati? Tutte le richieste di autorizzazione devono rappresentare accuratamente l'identità dell'applicativo integrato che cerca di accedere ai dati dell'utente. Se hai ottenuto le chiavi dell’utente per accedere ai servizi API di MadBit, mantieni queste chiavi riservate.
    • Quali dati vengono richiesti? È necessario fornire informazioni chiare e accurate che spieghino i tipi di dati richiesti. È necessario mantenere costantemente aggiornata la privacy policy e chiedere all'utente di acconsentire a qualsiasi modifica si voglia apportare alla tipologia di dati trattati o al trattamento stesso prima di porlo in essere.
    • Perché vengono richiesti i dati utente? È richiesta trasparenza con gli utenti quando viene spiegato lo scopo per cui l’applicativo integrato richiede i dati utente ed è vietato l’utilizzo degli stessi per scopi ulteriori. In generale, gli utenti dovrebbero essere in grado di comprendere prontamente il valore di fornire i dati richiesti dall'applicativo integrato nonché le conseguenze della condivisione di tali dati con l'applicativo integrato.

    2. Trasparenza e Privacy Policy

    È necessario pubblicare un'informativa sulla privacy che documenti completamente il modo in cui l’applicativo integrato interagisce con i dati dell'utente. È necessario rendere disponibile tale informativa quando l'applicazione viene resa disponibile al pubblico.

    L’informativa deve essere accurata, completa e facilmente accessibile.; descrivendo in modo preciso il modo in cui l’applicativo integrato accede, utilizza, archivia o condivide i dati utente.

    Prima di modificare il modo in cui l’applicativo integrato utilizza i dati utente è obbligatorio chiedere sempre l’assenso dell’utente.

    3. Minimizzazione e adeguatezza

    Le richieste di autorizzazione devono essere limitate alle sole informazioni necessarie per implementare l'applicativo integrato.

    Non va richiesto l'accesso a informazioni superflue e/o che non servono; l'accesso deve essere circoscritto all'ambito di accesso minimo e tecnicamente necessario per implementare funzionalità o servizi esistenti nell'applicativo integrato e l'accesso limitato alla quantità minima di dati necessari.

    4. Uso ingannevole

    È severamente vietato intraprendere qualsiasi attività che possa ingannare gli utenti o MadBit in merito all'utilizzo delle API di MadBit. Ciò include, a titolo esemplificativo, i seguenti requisiti:

    • Chiarezza su quali dati vengono raccolti e cosa viene fatto con essi. È necessario rendere noti tutti i dati utente ai quali si accede, che si utilizza, memorizza, elimina o che vengono condivisi, nonché qualsiasi azione intrapresa per conto di un utente. Non è consentito accedere, aggregare o analizzare i dati degli utenti di MadBit con il fine di visualizzarli, venderli o distribuirli in altro modo a terze parti. Nel complesso, non dovrebbero esserci sorprese per gli utenti di MadBit: funzionalità nascoste, servizi o azioni non coerenti con lo scopo commercializzato dell’applicativo integrato potrebbero indurre MadBit ad interdire l’accesso e l’utilizzo delle API
    • Divieto di utilizzo di API non documentate senza espressa autorizzazione. È vietato il reverse engineering delle API di MadBit non documentate o tentare in altro modo di derivare o utilizzare il codice sorgente sottostante alle API di MadBit non documentate. 
    • È vietato rilasciare dichiarazioni false o fuorvianti sull’identità di chi gestisce l’applicativo integrato. È necessario rappresentare accuratamente l'azienda, l'organizzazione o la persona che gestisce l'applicazione. Fornire false dichiarazioni sulla propria identità a MadBit o agli utenti di MadBit è motivo di interdizione all’accesso e utilizzo delle API.

    5. Sicurezza

    È necessario adottare misure ragionevoli e appropriate per proteggere tutte le applicazioni o i sistemi che utilizzano le API di MadBit contro l'accesso, l'uso, la distruzione, la perdita, l'alterazione o la divulgazione non autorizzati o illegali.

    6. Divieti di utilizzo

    È vietato utilizzare i dati delle API di MadBit per scopi pubblicitari, inclusa la pubblicità personalizzata, re-targeting e basata sugli interessi e la misurazione della pubblicità.

    È vietato utilizzare le API di MadBit per creare, addestrare o migliorare (direttamente o indirettamente) un prodotto o servizio simile o concorrente ad un software di titolarità di MadBit.

    È vietato accedere, memorizzare nella cache, copiare, analizzare o eseguire qualsiasi azione che coinvolga i dati dell'utente a meno che non sia espressamente consentito dall’utente stesso.

    È vietato effettuare attacchi informatici di ogni tipo verso le API di MadBit.

    7. Limiti di utilizzo

    È necessario sviluppare l’applicativo tenendo conto dei limiti di utilizzo specificati nella documentazione API. MadBit si riserva la possibilità di modificare questi limiti a propria discrezione.

    Il reiterato superamento di questi limiti è motivo di interdizione all’accesso e utilizzo delle API.

    8. Marchi e segni distintivi

    È possibile utilizzare il logo o i segni distintivi di MadBit e dei software di sua titolarità solo come prescritto nelle Brand Guidelines for FICdevs disponibili all’interno dell’area riservata allo Sviluppatore.
